# 本地个股 Markdown 存放规则

## 目录约定

个股分析 Markdown 统一存放在：

- `stock/`

不要再放在工作区根目录。

## 命名规则

统一使用：

- `股票名-YYYY-MM-DD.md`

示例：

- `隆平高科-2026-04-23.md`
- `金发科技-2026-04-21.md`

## 适用范围

以下内容放入 `stock/`：

- 个股财报分析
- 个股新闻与逻辑梳理
- 波段交易观察
- 按固定模板输出的股票 Markdown 报告

以下内容不要放入 `stock/`：

- 工作区说明文档
- memory/ 记忆文件
- 工具脚本说明
- 临时草稿
- 与个股分析无关的 Markdown

## 执行约定

以后新生成个股分析时：

1. 直接写入 `stock/`
2. 不在根目录落同名副本
3. 如需提交 GitHub，优先仅提交 `stock/` 下对应文件
